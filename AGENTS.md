# AGENTS

- 파일을 수정할 때는 각 소스 최상단의 수정 요약 주석도 함께 갱신합니다.
- 소스 코드 설명 주석은 한글로 작성합니다.
- OKX 봇과 업비트 봇은 가능한 한 같은 전략 구조를 유지합니다.
- 공통 전략 값은 `.env`에서 함께 관리합니다.
- 코인별 이격도, 익절률, 손절률도 `.env`에서 함께 관리합니다.
- 거래소별로 다른 값만 각 봇 또는 `.env`의 개별 항목으로 둡니다.
- 낮은 시드머니 테스트 상황을 항상 고려합니다.
- 최소 주문 금액과 수수료 때문에 주문이 과도하게 스킵되거나 과매매가 나지 않도록 주의합니다.
- 손절은 리스크 관리 우선 규칙으로 다루고, 일반 청산보다 우선되게 유지합니다.
- 신규 진입 품질을 높이는 필터와 청산 안전장치는 구분해서 설계합니다.
- 로그는 프로그램별로 분리해서 관리합니다.
- 분석용 로그와 운영 로그는 역할을 분리해서 관리합니다.
- 가능하면 system / strategy / trade 구조화 로그를 함께 남겨 집계 가능하게 유지합니다.
- 전략 로그에는 stage, reason 코드, 실제값, 기준값이 함께 남도록 유지합니다.
- 매수 신호와 매도 신호는 로그에서 바로 구분되도록 유지합니다.
- 설명 문서는 한국어 기준으로 간단하고 명확하게 정리합니다.
